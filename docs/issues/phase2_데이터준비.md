# Phase 2: 데이터 준비

## 작업 배경

CS 상담 데이터 분석 및 이슈 탐지를 위한 샘플 데이터를 생성합니다. 실제 쿠팡 CS 상담과 유사한 데이터 구조를 설계하고, 오안내 패턴이 포함된 100개 이상의 샘플 데이터를 생성하여 시스템 테스트 및 검증에 활용합니다.

## 작업 내용

### 1. 데이터 구조 설계
- Excel 파일 생성 (`cs_data.xlsx`)
- 필수 컬럼 정의:
  - 상담ID (고유 식별자)
  - 상담유형 (배송/결제/오배송/반품/교환 등)
  - 상담결과 (성공/오안내)
  - 통화시간(AHT) (초 단위)
  - VOC 키워드
  - 상담일시 (timestamp)
  - 상담사ID

### 2. 샘플 데이터 생성
- 최소 100개 이상의 샘플 데이터 생성
- 정상 케이스 데이터 (70%)
- 오안내 케이스 데이터 (30%)
- 특정 이슈 패턴 삽입:
  - 신선식품 반품 오안내 급증 패턴
  - 쿠팡이츠 배달 지연 오안내 패턴
  - 결제 오류 오안내 패턴

### 3. 데이터 검증
- 데이터 무결성 확인
- 컬럼별 데이터 타입 확인
- 결측치 확인 및 처리

## 인수 조건

- [ ] `data/cs_data.xlsx` 파일이 생성됨
- [ ] 모든 필수 컬럼이 정의되어 있음
- [ ] 최소 100개 이상의 샘플 데이터가 생성됨
- [ ] 정상:오안내 비율이 약 70:30으로 구성됨
- [ ] 3가지 이상의 특정 이슈 패턴이 포함됨
- [ ] 데이터 무결성 검증 완료 (결측치 없음, 타입 일치)
- [ ] Excel 파일을 열어 데이터 확인 가능
